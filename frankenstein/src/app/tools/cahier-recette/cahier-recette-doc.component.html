Cahier de Recette — Documentation

Système de tests de recette avec exécution IA

info 1. Vue d'ensemble

Le Cahier de Recette est une plateforme intégrée de gestion des tests de recette. Elle permet de définir des cas de test structurés, de les exécuter automatiquement via des agents IA (Claude ou Gemini), et de produire des rapports détaillés avec historique et analyses automatiques.

L'interface principale est organisée en 5 onglets :

login 2. Accès

  • radio_button_unchecked Bouton flottant teal (bas droite de l'écran) — visible sur toutes les pages si activé en configuration. Ouvre un popup affichant les tests liés à la page courante.
  • radio_button_unchecked Page /home — bouton "Cahier de recette" pour ouvrir directement le catalogue complet.
  • radio_button_unchecked L'accès peut être restreint aux administrateurs ou ouvert à tous les utilisateurs selon la configuration.

library_books 3. Onglet Catalogue

Catégories

Organisez vos tests par domaine fonctionnel (authentification, CRUD, navigation, formulaires, etc.). Chaque catégorie dispose d'un icône, d'une couleur et d'un ordre d'affichage.

Structure d'un test

Chaque cas de test contient :

  • Nom, priorité (critique / haute / normale / basse), statut (active / draft / deprecated)
  • Préconditions — état requis avant l'exécution
  • Étapes détaillées (voir format ci-dessous)
  • Pages cibles, tags, dépendances, durée estimée

Format des étapes

Chaque étape est définie sur 4 colonnes séparées par un pipe | :

Page | Action | Élément | Résultat attendu

Exemple :

/login | Saisir email | input[type=email] | Le champ accepte la valeur

campaign 4. Onglet Campagnes

  • Regroupements prédéfinis de tests pour des scénarios métier récurrents (ex : smoke test, test de régression).
  • Durée estimée automatique — calculée en additionnant les durées des tests inclus.
  • Lancement direct d'une campagne depuis l'onglet Campagnes ou depuis l'onglet Lancer.

play_circle 5. Onglet Lancer

Configuration du run

Avant de lancer, configurez : nom du run, URL du site, navigateur cible, environnement (local / staging / production) et le fournisseur IA.

I Estimation de durée — affichée avant le lancement en fonction des tests sélectionnés.
H Résolution des dépendances — les tests bloquants sont automatiquement ajoutés et exécutés en premier.
SSE Suivi en temps réel — progression et résultats partiels via Server-Sent Events.

bar_chart 6. Onglet Rapports

→ Score global — pourcentage de réussite (0–100%) calculé sur les tests non ignorés.
B Historique visuel (sparkline) — évolution du score sur les derniers runs.
C Comparaison entre deux runs — régressions, corrections et delta de score.
D Export — rapport complet au format JSON ou Markdown.
A Mode Replay — relance uniquement les tests ayant échoué lors du dernier run sélectionné.

settings 7. Onglet Avancé

G
Variables de contexte

Définissez des variables {{lb}}VAR_NAME{{rb}} substituées automatiquement dans les étapes au moment de l'exécution. Exemple : {{lb}}BASE_URL{{rb}} → https://app.example.com

E Bibliothèque de templates — importez des cas de test préconçus (auth, CRUD, navigation, formulaires, responsive, accessibilité).
J
Webhook CI/CD

Configurez une URL de webhook pour déclencher automatiquement un run depuis votre pipeline CI/CD.

F Tags — système d'étiquettes pour filtrer et regrouper les tests par thématique transversale.

widgets 8. Bouton flottant widget

Un bouton teal circulaire apparaît en bas à droite de chaque page (si activé en configuration). Il offre deux fonctionnalités directement contextuelles :

  • Tests enregistrés — affiche la liste des tests dont l'URL correspond à la page courante (targetPages).
  • Analyse IA — envoie le contenu de la page à l'IA pour suggérer automatiquement des cas de test pertinents, importables en un clic.

tune 9. Configuration

Widget flottant

Activez ou désactivez le bouton flottant depuis la page /config (section Cahier de Recette). Le paramètre est sauvegardé en base de configuration.

Variables d'environnement

Les variables de contexte sont définies dans l'onglet Avancé et peuvent être personnalisées par run.
